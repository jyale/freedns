Welcome to the FreeDNS project page.

Project Overview

The FreeDNS project is a Yale University computer science collaboration between John Maheswaran, Dr Ramki Gummadi, Prof Bryan Ford and Prof Avi Silberschatz.

Firefox Extension

To access FreeDNS you need to add the Firefox extension. Click the button below to get it.

Add to Firefox!

You may be prompted to allow the extension to be added to Firefox in which case you must click "Allow".

Once you've installed the plugin, you will see a menu item in Firefox that allows you to turn on and turn off access to FreeDNS (see picture below). Clicking "Toggle FreeDNS access" will turn access to FreeDNS on and off.



Once installed, you will be able to access the FreeDNS world, that is addresses that end in the suffix ".freedns" such as 6f700b83be72c6e24c45612e04717103.freedns . Simply navigate to that page in your web browser. Valid FreeDNS names are 32 character hexadecimal strings (ssh-keygen public key fingerprints) or 40 character hexadecimal strings (SHA-1 hashes), suffixed by ".freedns" such as 6f700b83be72c6e24c45612e04717103.freedns . Note that FreeDNS is an ongoing project still under development and all valid .freedns addresses currently redirect to dedis.cs.yale.edu and all invalid .freedns names such as invalidname.freedns redirect to Google.

Phone number domains

We have also been working on a feature that allows you to register an IP address to any cellphone number which is then accessible through FreeDNS. This allows users to be able to look up, for example a person or organization's homepage knowing only their phone number without having to worry about the page being a phishing, spoof or otherwise unofficial or unaffiliated site. For example the phone number (203)781-4585 has been registered to IP address 130.132.35.53 (the Yale website) which is then accessible by visiting 2037814585.freedns. The IP address can be changed by the cellphone owner.

MAP and ADD services (as of 7/25/2013)

We have implemented a backend database running on MySQL which stores records of the form:

(URL, CID, IP, title)

CID = cryptographic ID (ie SHA-1 hash, a FreeDNS name).

The lookup service now on receiving a FreeDNS request, looks it up in the database to see if there is an entry and if so returns that IP address.

We have also implemented an ADD service for registering new FreeDNS names which is now available online.
